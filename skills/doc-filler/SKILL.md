# SOP Doc Filler

自動從後台頁面生成 SOP 操作步驟並填充到 Zensical 文件中。

## 使用時機

當用戶說：
- `@doc.md #heading` - 填充特定章節
- 「填充這個章節的 SOP 內容」
- 「幫我生成這個功能操作步驟」
- 「從後台頁面擷取操作流程」

## 工作流程

### 1. 解析輸入

從用戶輸入中提取：
- **文件路徑**：`@` 後的路徑（如 `docs/ec/orders/編輯訂單內容.md`）
- **標題名稱**：`#` 後的標題（如 `新增自訂品項`）

### 2. 讀取文件並定位區塊

1. 使用 `read` 工具讀取目標文件
2. 找到對應的 heading（如 `### 新增自訂品項/折扣 {#edit-order-custom}`）
3. 確認該區塊是否已存在內容

### 3. 訪問後台頁面並擷取資訊

1. 使用 `chrome-devtools_navigate_page` 或 `chrome-devtools_select_page` 訪問對應後台頁面
2. 使用 `chrome-devtools_take_snapshot` 獲取頁面結構
3. 根據 heading 名稱，識別需要操作的功能區域：
   - **自訂品項**：尋找「新增自訂品項」按鈕，點擊後截取彈窗
   - **自訂折扣**：尋找「新增自訂折扣」連結，點擊後截取彈窗
   - **商家備註**：尋找「商家備註」文字框區域
   - **新增商品**：尋找「新增商品」按鈕，點擊後截取彈窗
   - **增減或移除商品**：尋找商品列表區域

### 4. 生成操作步驟

根據頁面結構，生成標準化步驟：

```markdown
1. 點選「[按鈕名稱]」[按鈕/連結]，開啟設定彈窗。
2. 填寫以下必填欄位：
    - **欄位1**：說明（例：範例內容）。
    - **欄位2**：說明（例：範例內容）。
3. 點選「確認」將 [項目] 加入訂單。
4. 操作完成後點擊「確認編輯」，並在二次彈窗中點選「確認並送出」即可套用變更。

![截圖描述](../../assets/images/[截圖檔名].png)
```

### 5. 截取並儲存截圖

1. 使用 `chrome-devtools_take_screenshot` 截取對應區域：
   - 對於彈窗：截取整個彈窗
   - 對於頁面區域：截取包含標題和功能的區域
2. 儲存到 `docs/assets/images/` 目錄
3. 檔名格式：`EC-[頁面名稱]-[功能名稱].png`

### 6. 更新文件

使用 `edit` 工具更新文件：
1. 找到對應的 heading 區塊
2. 如果存在 `=== "選項"` 的 tabs 格式，在對應 tab 下填充內容
3. 如果不存在 tabs，直接在 heading 下填充內容
4. 添加截圖引用
5. 更新 `last_modified` 時間

### 7. 驗證

1. 確認截圖已正確儲存
2. 確認 frontmatter 的 `last_modified` 已更新
3. 確認步驟格式符合 AGENTS.md 規範

## 注意事項

- **截圖最小化原則**：只截取功能相關的最小區域，避免截取整個頁面
- **步驟標準化**：所有操作步驟都應包含「點選按鈕 → 填寫欄位 → 確認 → 儲存」的完整流程
- **必填欄位**：務必標註哪些欄位是必填的
- **二次彈窗**：大多數編輯操作都需要二次確認，步驟中要包含此說明
