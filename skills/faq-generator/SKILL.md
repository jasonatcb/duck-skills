# FAQ 生成技能

此技能會根據文檔內容生成常見問題 (FAQ) 並添加到文件中。

## 使用方式

用戶提供文檔路徑作為參數，例如：`為 docpath.md 添加 FAQ`

## 執行步驟

### 1. 讀取文檔內容
- 讀取用戶指定的文檔路徑
- 分析文檔的主要內容、重點功能、設定項目

### 2. 生成相關的常見問題
根據文檔內容，生成 2-3 個合理的常見問題：
- 問題應該與文檔主題直接相關
- 答案應該基於文檔中提到的功能或設定
- 避免生成文檔未提及的內容

### 3. 添加 FAQ 到文檔
使用 zensical quote callout 格式：

```markdown
## 常見問題

??? quote "問題文字"
    [](){ #faq-slug }

    答案內容
```

**格式規則：**
- 使用 `??? quote` 開頭，問題文字放在雙引號內
- 下一行為 `[](){ #slug }` 錨點（**無空行**），不再使用 `####` 重複標題
- 錨點 slug 使用 kebab-case 英文，格式 `faq-{文檔主題slug}-{問題關鍵字}`（如文件 slug 為 `tcat-home`，則 `#faq-tcat-home-download-no-response`）
- 錨點與答案內容之間**需空一行**
- 答案內容縮排 4 個空格
- 每個 FAQ 之間用空行分隔

### 4. 更新 last_modified
如果文檔有 frontmatter，請更新 `last_modified` 欄位為當前時間（格式：YYYY-MM-DD HH:mm）。

## 範例

**輸入文檔：** `docs/ec/payments/設定信用卡.md`
（內容關於如何設定信用卡付款方式）

**輸出：**

```markdown
## 常見問題

??? quote "支援哪些信用卡種類？"
    [](){ #faq-credit-card-setup-card-types }

    目前支援 Visa、Mastercard、JCB、美國運通等主流信用卡。

??? quote "分期付款最多可以分幾期？"
    [](){ #faq-credit-card-setup-installment-periods }

    最長可支援 12 期分期，具体期数视银行方案而定。
```

## 注意事項

- 如果文檔中已有 `## 常見問題` 區塊，在現有區塊下方新增
- 如果沒有 `## 常見問題` 區塊，在文檔末尾（frontmatter 之後）新增
- **標題格式：** 統一使用 `## 常見問題`，不要使用 `## 常見問題 FAQ`、`## 常見問題(FAQ)` 或其他變體
- **禁止編號前綴：** 問題文字中不要加「Q1.」、「Q2.」等編號前綴
- **格式一致化：** 如果原文件已有其他 callout 格式（如 `??? question`），仍需將新 FAQ 統一使用 `??? quote` 格式，不要混用
- 確保每個 FAQ 的答案簡潔明瞭，不超過 3-4 個要点
- **清單格式**：若答案中需要列出檢查項目，使用 checklist 格式 `- [x]`，並在清單上方留空行
- **有序列表**：若為步驟說明，使用 `1.` 格式，勿使用 `1)`
- **粗體格式**：使用 `**文字**` 時，**前後需加空格**以確保正確 render，如 `** 實體店面 **`
- **無序/有序清單**：在 quote callout 內使用清單時，清單上方必須留一行空白，否則無法正確 render
- **內部連結**：FAQ 答案中提及的專有名詞或操作，若有對應的文件，應加入 title 連結：
  - 格式：`[術語](對應文件.md#錨點){ title="目標頁面標題" }`
  - 例如：`[首次儲值](建立 Meta 廣告帳號並儲值.md#儲值廣告金){ title="建立 Meta 廣告帳號並儲值" }`
  - 參考資料區段的連結不需要 title 屬性，使用純連結即可

## 無序清單範例

```markdown
??? quote "為什麼看不到搜尋關鍵字資料？"
    [](){ #faq-keyword-search-data-not-visible }

    可能原因包括：

    - 原因一說明
    - 原因二說明
    - 原因三說明
```

## 有序清單範例

```markdown
??? quote "如何設定站內搜尋？"
    [](){ #faq-keyword-search-setup }

    請依序執行以下步驟：

    1. 第一步說明
    2. 第二步說明
    3. 第三步說明
```
